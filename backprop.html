A tiny network — two inputs, two hidden neurons, two outputs — learns from a single example. Every number below is computed live, not pre-baked, so you can change the inputs and watch the whole trace update.

Forward pass — signal moves left to right Backward pass — error moves right to left Weight update — one gradient-descent step

A network learns by repeating four moves: push data forward to get a prediction, measure how wrong that prediction is, push the error backward through the network to work out which weights caused it, then nudge every weight a little in the direction that reduces the error. That backward move — using the chain rule to assign each weight its share of the blame — is backpropagation. Click through the steps below to watch it happen with real numbers.

Reference

Key terms

Forward pass
Pushing an input through the network, layer by layer, to produce a prediction.
Loss
A single number measuring how far the prediction is from the target. This example uses sum of squared errors.
Gradient (∂L/∂w)
How much the loss would change if one weight moved slightly — the direction and size of a weight's "blame."
Chain rule
The calculus rule that lets a gradient be computed as a product of simpler local derivatives, layer by layer.
Backpropagation
Applying the chain rule from the output backward to the input to get every weight's gradient in one pass.
Sigmoid, σ(x)
The activation function 1/(1+e⁻̀ ), squashing any number into the range (0, 1).
Learning rate, η
How large a step each weight takes against its gradient during an update.
Epoch
One full cycle of forward pass, loss, backward pass and weight update.
