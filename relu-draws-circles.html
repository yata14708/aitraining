How ReLU draws bumps and circles

Every kink below is one neuron switching on. Flip the weight's sign and the "up" ramp becomes a "down" ramp — that's the whole trick.

Positive weight

ReLU(x) — flat, then rises

Negative weight

ReLU(−x) — falls, then flat

Three neurons → A, B, C

ReLU(x) − 2·ReLU(x−1) + ReLU(x−2)

A (x=0): slope turns +1, starts rising. B (x=1): slope flips to −1, this is the peak. C (x=2): slope returns to 0, back at baseline. Two neurons only ramp-then-plateau — it takes a third hinge to come back down.

So how does it actually close?

A single graph can't loop — the input always moves left to right. The fix: use two outputs, x(t) and y(t), each an ordinary hat-shaped graph in a hidden variable t. Neither one loops on its own. But if x lands back where it started, and y lands back where it started, then plotting the pair (x,y) together — and forgetting t existed — traces a path that returns to its start point. That return is the "close."

y(t) — never loops

Both start and end dots sit at the same height — same x, same y. That's the whole trick.

Drop t, keep only (x,y): the triangle closes, even though t itself only ever ran forward.
